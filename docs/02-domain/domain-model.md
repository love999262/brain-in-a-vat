# 领域模型

## 目的

本文档定义引擎内部的核心领域对象，以及它们之间的关系。

## 核心对象

### `World`

一组角色、消息源、关系和场景组成的运行世界。

关键职责：

- 持有世界级配置
- 管理运行生命周期
- 汇总世界级状态

### `Character`

角色模板或角色实例。模板来自 Character Pack，实例由 World Pack 装配。

关键职责：

- 持有人格画像
- 维护实例态
- 参与互动、记忆和关系变化

### `Relationship`

角色之间的有向关系边。

关键职责：

- 记录熟悉度、信任、亲近、张力、好奇、影响等维度
- 随互动、消息源和用户行为动态变化

### `Memory`

角色和世界的记忆结构。

关键职责：

- 保留短期上下文
- 沉淀情节事件
- 保留长期认知

### `Scene`

当前世界的前台表现和互动上下文。

关键职责：

- 定义可见角色
- 定义默认焦点
- 定义频道和场景展示模式

### `Source`

外部信息输入来源。

关键职责：

- 提供条目
- 提供分类和可信度线索
- 提供源自身过滤和规范化信息

### `Interaction`

一次消息、回应、打断、争论、认同、旁观行为。

关键职责：

- 作为记忆和关系变化的触发单元
- 连接用户、角色、消息源和场景

### `Event`

系统内外统一的状态变化单位。

关键职责：

- 对内作为调试和回放基础
- 对外作为宿主和 UI 的订阅对象

## 关系图

- World 包含 Character 实例
- Character 之间通过 Relationship 连接
- Interaction 会写入 Memory 并影响 Relationship
- Source 通过消费管线进入 Interaction 或 Memory
- Scene 决定当前谁被看见、谁被聚焦、谁在当前频道中活动

## 设计约束

- 模板和实例必须分开
- 记忆和关系都是运行时状态，不属于 Pack 静态内容
- UI 不直接使用内部领域对象
