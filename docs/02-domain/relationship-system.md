# 关系系统

## 目标

关系系统负责描述角色之间如何互相影响，以及这种影响如何随时间变化。

## 基础结构

关系是有向的。  
`A -> B` 与 `B -> A` 是两条不同的边。

## V1 固定维度

- `familiarity`：熟悉度
- `trust`：信任
- `warmth`：亲近感
- `tension`：张力或分歧
- `curiosity`：关注与兴趣
- `influence`：受对方影响的倾向

## 初始来源

关系初值由两个部分叠加：

- Character Pack 中的默认关系画像
- World Pack 中的显式初始关系

世界内的显式关系优先于默认画像。

## 变化来源

关系变化主要来自：

- 用户偏向某个角色
- 角色直接互动
- 对同一信息的相似或相反反应
- 共享记忆中出现高权重事件

## 更新原则

- 单次变化不应过大
- 长期一致行为应能形成稳定偏移
- 冲突和亲近可以同时存在
- 高张力不等于低熟悉度

## 输出要求

关系系统至少要对外提供：

- 主要关系边
- 最近变化摘要
- 可供 UI 展示的关系视图数据

## 边界

- UI 不参与关系计算
- Renderer 不读取完整关系矩阵
- Pack 只能提供默认值和提示，不能覆盖运行中关系变化
