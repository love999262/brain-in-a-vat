# 记忆系统

## 目标

记忆系统负责在有限资源下保留角色成长和世界演化所需的信息，同时避免无脑堆积。

## 记忆分层

### 短期记忆

用途：

- 保存最近几轮对话
- 保存当前正在消费的消息源片段
- 为当前生成提供上下文

### 情节记忆

用途：

- 记录重要互动事件
- 记录用户影响和角色之间的关键节点
- 作为中期演化依据

### 长期记忆

用途：

- 保留稳定偏好
- 保留长期关系变化
- 保留世界观和重复出现的重要结论

## 写入流程

1. 输入进入系统
2. 判定来源和相关角色
3. 计算显著性
4. 写入短期记忆
5. 必要时生成情节摘要
6. 满足条件时提升为长期记忆

## 显著性因子

- 与角色自我相关
- 与用户相关
- 情绪强度
- 新颖度
- 重复暴露程度
- 来源可信度
- 与当前世界主题的相关度

## 记忆压缩

V1 需要保留压缩入口，至少支持：

- 去重
- 归并重复主题
- 原文转摘要
- 低价值记忆降权

## 审计能力

每条记忆至少应保留：

- 来源
- 创建时间
- 影响角色
- 关联事件
- 当前层级

## 删除与回滚

记忆系统必须支持：

- 指定记忆删除
- 按范围删除
- 降权
- 回滚到导出快照

## 边界

- 记忆偏好由 Character Pack 描述
- 记忆算法和保留策略由 Engine 决定
