# 存储与持久化

## 目标

默认采用本地优先持久化，保证世界状态、记忆和关系在浏览器端可恢复。

## 分层

### `IndexedDB`

用于：

- 世界状态
- 角色状态
- 记忆
- 关系
- 日志
- 消费游标

### `OPFS` 或等价大文件层

用于：

- 模型文件
- 大资源缓存

## 命名空间

持久化必须支持命名空间，以区分：

- 不同世界
- 不同用户实例
- 不同环境

## 恢复

支持：

- 自动恢复上次状态
- 从快照导入
- 恢复失败后的安全降级

## 边界

存储 Provider 负责“怎么存”，引擎决定“存什么”和“什么时候存”。
